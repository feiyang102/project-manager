# 产品定义

> 所属 PRD：[本地项目与文件夹管理后台 PRD](../PRD.md)

## 1. 文档信息

- 产品名称：本地项目与文件夹管理后台
- 产品类型：本地运行的可视化管理后台
- 目标版本：V1.0
- 目标用户：个人用户，主要用于管理本机收藏文件、常用文件夹、开发项目、资料项目和长期事项
- 运行环境：本地电脑，优先支持 macOS，后续可扩展 Windows / Linux
- 技术方向：Next.js 全栈应用，本地数据库，本地文件系统访问能力

## 2. 产品背景

用户在本地电脑中长期积累了大量项目、文件夹、文档、素材、代码仓库、灵感资料和工作记录。它们分散在不同路径、不同命名规则和不同工具中，导致查找困难、项目状态不清晰、资料无法持续归档。

本产品希望提供一个运行在本地的可视化管理后台，帮助用户把"文件夹"和"项目"统一管理起来。它不是网盘，也不是团队协作平台，而是一个面向个人工作流的本地知识与项目资产管理中心。

## 3. 产品目标

1. 让用户可以把本地文件、文件夹、代码项目、资料目录加入收藏，并以可视化方式管理。
2. 让每个项目拥有独立记录，包括名称、路径、状态、说明、标签、关联文件、笔记和关键时间。
3. 支持快速搜索、筛选、分类和查看最近使用内容，减少在 Finder、终端和编辑器之间反复查找的成本。
4. 保持数据本地化，用户的文件路径、项目记录和元数据默认只存储在本机。
5. 为后续扩展自动扫描、AI 摘要、任务管理、备份同步、代码仓库状态检测等能力预留结构。

## 4. 非目标范围

V1.0 不做以下能力：

- 不做多人协作、组织空间、权限审批。
- 不做云端文件同步或网盘替代。
- 不直接修改用户文件内容，除非用户明确执行打开、重命名、移动等高风险操作。
- 不做完整项目管理系统中的甘特图、复杂排期和团队工时。
- 不做在线账号体系，默认无需登录。
