# 交互视觉与技术建议

> 所属 PRD：[本地项目与文件夹管理后台 PRD](../PRD.md)

## 12. 交互与视觉要求

- 整体风格应偏管理后台，信息密度适中，避免营销页风格。
- 左侧导航常驻，主区域展示列表、表格、详情和操作面板。
- 状态、类型和标签需要有清晰视觉区分。
- 高风险操作需要二次确认，例如删除项目、删除收藏项。
- 路径失效、数据为空、搜索无结果时需要有明确空状态。
- 表单保存后需要有成功或失败反馈。
- 支持浅色主题，后续可扩展深色主题。

## 14. 技术建议

### 14.1 前端

- Next.js App Router
- TypeScript
- Tailwind CSS
- shadcn/ui 或自建轻量组件
- lucide-react 图标

### 14.2 后端与本地能力

- Next.js Route Handlers / Server Actions
- SQLite 作为本地数据库
- Prisma 或 Drizzle ORM
- Node.js `fs` 模块读取文件元数据
- 通过系统命令或桌面容器能力打开本地路径

### 14.3 后续桌面化方向

如果浏览器环境对本地文件系统访问限制较多，可以考虑：

- Electron
- Tauri
- macOS 本地脚本桥接
